# Support

If you need help with EnvForage, you're in the right place!

## Where to get help

### 1. Documentation
Always check the documentation first. The `docs/` folder contains extensive details on architecture, features, workflows, and API design.
- [Features and Status](./docs/FEATURES.md)
- [Workflow Guide](./docs/WORKFLOW.md)
- [API Design](./docs/API_DESIGN.md)

### 2. GitHub Issues
If you encounter a bug or have a feature request, please [open an issue](https://github.com/rishabh0510rishabh/EnvForage/issues). We use issue templates to gather necessary information, so please fill them out completely.

- **Bug Reports**: Please include your OS, EnvForage version, and the exact steps to reproduce the issue. Running `envforage diagnose` and providing the output helps immensely.
- **Feature Requests**: Describe the problem you're trying to solve and how the proposed feature would help.

### 3. Community Discussions
For general questions, discussions about best practices, or showcasing how you use EnvForage, please use [GitHub Discussions](https://github.com/rishabh0510rishabh/EnvForage/discussions).

## Security Issues
If you find a security vulnerability, please do NOT open a public issue. Review our [Security Policy](./SECURITY.md) and report it to `rishabh0510@gmail.com`.
